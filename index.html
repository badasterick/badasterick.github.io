Robot Ball



Overview:

Robot Ball is a time based puzzle/platformer game where you play as a robot that is about to self destruct. You must complete the level before this happens. The robot can be damaged by lasers and other traps which will speed up the self destruction.

Role:

I independently designed and developed this game. It consists of two levels and a bonus level that uses different stationary cameras that will invert the players controls. The environment assets are from the Scifi Hallway pack on the Unreal Marketplace. The BB8 character model and audio assets were also downloaded from free to use websites. I created the skysphere using a software called Skyscape.

Medical Simulations


Overview:

I professionally develop VR/AR surgery simulations. Many of these simulations use a haptic pen which allows surgeons to feel objects in 3D space. The goal of these simulations is to allow doctors a chance to practice surgeries in virtual space before the actual surgery occurs.

Role:

As Technical Designer, I develop Virtual Reality simulations that allow doctors to interact with real patient data in 3D space. I am responsible for the design and development of the simulations. Additionally I manager the User Experience and create videos to be used as marketing content.


  • Date: May 2018
  • Software: Unity
  • Category: Simulation

Fighter Project


Overview:

This project was created as a side project as part of a Developer Support & Documentation internship at Epic Games. It's purpose was to demonstrate and document the steps to make a fighting game quickly using the Unreal Engine. This is a two player fighting game where each player has three types of punches and a block that they can use. The types of punches work with a rock, paper, scissors style mechanic: Jab beats Uppercut, Uppercut beats Hook, and Hook beats Jab.

Role:

I was responsible for the design, development, and documentation for this project. Working with the employees at Epic Games, I recorded reference footage which was used to make the animations in the project realistic.

  • Date: August 2015
  • Software: Unreal Engine 4
  • Category: Games

GrayScale


Overview:

GrayScale plays as if you were playing a tower defense game on a pad of paper. Stick figure enemies with different abilities will appear from outside of the screen to try to damage your player. Use the mouse as if it were a pencil and erase the enemies. You can also use the mouse to "draw" obstacles to help prevent the enemy advances. There are ten levels total and a boss battle to win the game.

Role:

My primary roles were as project manager and game designer. Our team consisted of five people with an eight week development cycle. I also worked on the sound, UX desgin, and opening sequence for the game.

  • Date: April 2013
  • Software: XNA

Arduino Flyer


Overview:

This game is an endless runner where the player navigates a spaceship through procedurally generated obstacles and enemies. For input, an ultrasonic sensor will detect the players hand position and move the spaceship to match its movements. A 3D printed joystick and rail system works with the sensor to create an arcade like experience.

Role:

I designed and developed this project in its entirety over five weeks. The goal of this project was for me to find new ways to integrate other technologies into a video game. Through this project I learned how to develop electronic prototypes using an Arduino as well as how to 3D print custom made assets. The spaceship 3D model was downloaded from a free to use website.


  • Date: May 2016
  • Software: Unreal Engine 4
  • Category: Games

Pixalto


Overview:

Pixalto is a 2D platformer that plays on the idea of defining character. The player will traverse the levels and interact with different types of NPCs that may be friendly or hostile. The player decides what lives and what dies. The catch is if the player chooses to kill an NPC, they must literally throw away a part of themselves to do so. Allowing NPCs to live allows the player to add new parts to their character. I researched and documented my designs into a paper that can be downloaded with the project below.

Role:

For Pixalto, I worked as a game designer and artist on this project. My main focus was to demonstrate the idea of visualizing a person's character through gameplay. This project was completed in three weeks by a team of five developers including myself.
